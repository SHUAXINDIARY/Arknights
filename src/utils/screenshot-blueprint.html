入口：ShowResult.handleSave()
主链路：savePngByHtmlInCanvas(targetElement)
回退链路：savePngByCanvas(isDown)
输入 / 上下文
来源页面
ShowResult.tsx
`handleSave` 先调用新链路，再根据布尔结果决定分支。
目标节点
targetElement
通常是结果容器（`ref.current`），用于限制截图范围。
约束条件
浏览器能力
依赖实验性 drawElementImage API。
主链路：savePngByHtmlInCanvas
阶段 1
能力门禁
`hasHtmlInCanvasSupport()` 检查上下文方法是否可用。
阶段 2
离屏构建
创建隐藏 host + `canvas[layoutsubtree]` + DOM 克隆子树。
阶段 3
样式对齐修正
补齐继承文本度量，并将 host 挂载到父级上下文。
阶段 4
图片内联修正
`inlineCloneImages`：远程图片 -> blob -> data URL，并等待 ready。
阶段 5
绘制调度
等待 2 帧，监听 `onpaint`，触发 `requestPaint`，保留 timeout 兜底。
阶段 6
栅格化与导出
clear + 填充背景 + scale + drawElementImage + blob 下载。
返回约定
仅当完整链路成功时返回 true。
遇到 API 不支持、跨域图片未消解或绘制异常时返回 false。
缓存
IMAGE_DATA_URL_CACHE
避免同一远程 URL 重复转换。
输出
Arknights.png
由 `saveAs(blob, EXPORT_FILENAME)` 生成。
回退 / 旧链路
分支
if primary === false
调用方会立即切换到旧链路。
旧链路
savePngByCanvas
`domtoimage.toSvg` -> Image decode -> OffscreenCanvas draw -> saveAs。
苹果路径
isApple + waitFrames
最终下载前延迟若干帧，降低 Safari 时序问题影响。
可靠性
用户可见保障
优先现代链路，并在失败时保持导出可用性。
实现对比 / 扩展能力 / 上限 / 性能
对比维度
新链路（html-in-canvas）
旧链路（dom-to-image + OffscreenCanvas）
扩展能力
绘制前可插入更多钩子：样式修正、图片内联、帧调度与分阶段观测。
固定短链路（toSvg -> drawImage -> blob），可介入点更少。
渲染上限
对克隆 DOM 上下文与继承排版可控性更高，支持绘制前资源治理。
更偏通用兜底；中间经过 SVG 栅格化，对特定偏差的精细修正能力有限。
导出范围
通过传入 targetElement 可天然支持局部导出。
当前实现对 document.body 快照，成本与整页复杂度强耦合。
性能画像
缓存命中与局部渲染时更有优势；远程图转换与多阶段等待会引入额外成本。
简单页面下编排开销更低；整页 DOM/SVG 规模上升时退化明显。
优化空间
后续方向明确：fetch 并发上限、重试策略、阶段耗时指标与失败遥测。
主要集中在快照规模与下游 canvas 阶段的粗粒度优化。
